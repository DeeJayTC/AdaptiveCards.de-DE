## <a name="how-they-work"></a><span data-ttu-id="f9016-104">Funktionsweise</span><span class="sxs-lookup"><span data-stu-id="f9016-104">How they work</span></span>

<span data-ttu-id="f9016-107">Contoso Bot kann beispielsweise eine adaptive Karte über Bot Framework erstellen, die bei der Übermittlung an Skype das Erscheinungsbild einer Skype-Karte annimmt.</span><span class="sxs-lookup"><span data-stu-id="f9016-107">For example, Contoso Bot can author an Adaptive Card through the Bot Framework, and when delivered to Skype, it will look and feel like a Skype card.</span></span> <span data-ttu-id="f9016-108">Wenn die gleiche Nutzlast an Microsoft Teams gesendet wird, nimmt die das Erscheinungsbild von Microsoft Teams an.</span><span class="sxs-lookup"><span data-stu-id="f9016-108">When that same payload is sent to Microsoft Teams, it will look and feel like Microsoft Teams.</span></span> <span data-ttu-id="f9016-109">Da immer mehr Host-Apps beginnen, Adaptive Karten zu unterstützen, wird dieselbe Nutzlast automatisch in diesen Anwendungen bereitgestellt, wobei die Darstellung wie speziell für die jeweilige App konzipiert erscheint.</span><span class="sxs-lookup"><span data-stu-id="f9016-109">As more host apps start to support Adaptive Cards, that same payload will automatically light up inside these applications, yet still feel entirely native to the app.</span></span>

<span data-ttu-id="f9016-110">Benutzer haben den Vorteil, dass sich alles vertraut anfühlt.</span><span class="sxs-lookup"><span data-stu-id="f9016-110">Users win because everything feels familiar.</span></span> <span data-ttu-id="f9016-111">Hosts haben den Vorteil, die Benutzerfreundlichkeit steuern zu können.</span><span class="sxs-lookup"><span data-stu-id="f9016-111">Host apps win because they control the user experience.</span></span> <span data-ttu-id="f9016-112">Und Kartenautoren haben den Vorteil, dass ihre Inhalte ohne zusätzlichen Aufwand eine größere Reichweite erhalten.</span><span class="sxs-lookup"><span data-stu-id="f9016-112">And Card Authors win because their content gets broader reach without any additional work.</span></span>

## <a name="for-card-authors"></a><span data-ttu-id="f9016-121">Für Kartenautoren</span><span class="sxs-lookup"><span data-stu-id="f9016-121">For Card Authors</span></span>
<span data-ttu-id="f9016-122">Adaptive Karten eignen sich hervorragend für Kartenautoren:</span><span class="sxs-lookup"><span data-stu-id="f9016-122">Adaptive Cards are great for card authors:</span></span>

* <span data-ttu-id="f9016-124">**Umfangreicherer Ausdruck**:Ihr Inhalt kann besser daran ausgerichtet werden, was Sie sagen möchten, da Sie über eine reichhaltigere Palette an Nuancen verfügen.</span><span class="sxs-lookup"><span data-stu-id="f9016-124">**Richer expression** - Your content can more closely align with want you want to say because you have a richer palette to paint with.</span></span>

* <span data-ttu-id="f9016-130">**Einheitliche Benutzerumgebung**: Sie garantieren eine einheitliche Umgebung für Ihre Benutzer, da Sie die Formatvorlagen der gerenderten Karte besitzen.</span><span class="sxs-lookup"><span data-stu-id="f9016-130">**Consistent user experience** - You guarantee a consistent experience for your users, because you own the style of the rendered card.</span></span>

* <span data-ttu-id="f9016-132">**Sicher**: Inhalte werden über sichere Nutzlasten bereitgestellt, sodass Sie Ihr Benutzeroberflächenframework nicht für Rohmarkup und -skripting öffnen müssen.</span><span class="sxs-lookup"><span data-stu-id="f9016-132">**Safe** - Content is delivered in safe payloads so you don't have to open up your UI framework to raw markup and scripting.</span></span>

## <a name="core-design-principles"></a><span data-ttu-id="f9016-136">Wichtigste Designprinzipien</span><span class="sxs-lookup"><span data-stu-id="f9016-136">Core Design Principles</span></span> 

<span data-ttu-id="f9016-137">Adaptive Karten werden durch eine Reihe von [Leitprinzipien](resources/principles.md) gesteuert, die sich für die Einhaltung des Designs als nützlich erwiesen haben.</span><span class="sxs-lookup"><span data-stu-id="f9016-137">Adaptive Cards are driven by a set of [guiding principles](resources/principles.md) that have been useful for keeping the design on track.</span></span> 

### <a name="semantic-instead-of-pixel-perfect"></a><span data-ttu-id="f9016-138">Semantik statt Pixelpräzision</span><span class="sxs-lookup"><span data-stu-id="f9016-138">Semantic instead of pixel-perfect</span></span>
<span data-ttu-id="f9016-139">Wir haben uns beim Layout so stark wie möglich um semantische Werte und Konzepte im Gegensatz zu reiner Pixelperfektion bemüht.</span><span class="sxs-lookup"><span data-stu-id="f9016-139">We have striven as much as possible for semantic values and concepts as opposed to pure pixel-perfect layout.</span></span> <span data-ttu-id="f9016-140">Beispiele für semantische Ausdrücke finden sich in Farben, Größen und Elementen wie „FactSet“ und „ImageSet“.</span><span class="sxs-lookup"><span data-stu-id="f9016-140">Examples of semantic expression show up in colors, sizes, and in elements like FactSet and ImageSet.</span></span> <span data-ttu-id="f9016-141">Diese ermöglichen es der Host-Anwendung, bessere Entscheidungen über das tatsächliche Erscheinungsbild zu treffen.</span><span class="sxs-lookup"><span data-stu-id="f9016-141">These all allow the host application to make better decisions about the actual look and feel.</span></span>

### <a name="card-authors-own-the-content-host-app-owns-the-look-and-feel"></a><span data-ttu-id="f9016-142">Kartenautoren besitzen den Inhalt, die Host-App das Erscheinungsbild.</span><span class="sxs-lookup"><span data-stu-id="f9016-142">Card Authors own the content, Host App owns the look and feel</span></span>
<span data-ttu-id="f9016-143">Die Kartenautoren besitzen ihre beabsichtigte Aussage, die anzeigende Anwendung besitzt jedoch Erscheinungsbild der Karte im Kontext ihrer Anwendung.</span><span class="sxs-lookup"><span data-stu-id="f9016-143">The card authors own what they want to say, but the application displaying it owns the look and feel of the card in the context of their application.</span></span>

### <a name="keep-it-simple-but-expressive"></a><span data-ttu-id="f9016-144">Halten Sie es einfach, aber ausdrucksstark.</span><span class="sxs-lookup"><span data-stu-id="f9016-144">Keep it simple, but expressive</span></span>
<span data-ttu-id="f9016-145">Wir möchten, dass Adaptive Karten ausdrucksstark und universell einsetzbar sind, möchten aber kein Benutzeroberflächenframework erstellen.</span><span class="sxs-lookup"><span data-stu-id="f9016-145">We want Adaptive Cards to be expressive and general purpose, but we don't want to build a UI framework.</span></span>  <span data-ttu-id="f9016-146">Das Ziel ist die Erstellung einer Zwischenebene, die „aussagekräftig genug“ ist, so wie Markdown aussagekräftig genug für Dokumente ist.</span><span class="sxs-lookup"><span data-stu-id="f9016-146">The goal is to create an intermediate layer which is "expressive enough" in the same way Markdown is expressive enough for documents.</span></span>

<span data-ttu-id="f9016-147">Durch den Schwerpunkt auf Einfachheit und Aussagekraft hat Markdown eine einfache und konsistente Beschreibung von Dokumentinhalt erstellt.</span><span class="sxs-lookup"><span data-stu-id="f9016-147">By focusing on keeping it simple and expressive, Markdown created an easy and consistent description of document content.</span></span>  <span data-ttu-id="f9016-148">Wir sind der Meinung, dass Adaptive Karten auf die gleiche Weise ein einfaches, ausdrucksstarkes Mittel zur Beschreibung von Karteninhalten darstellen können.</span><span class="sxs-lookup"><span data-stu-id="f9016-148">In the same way, we believe that Adaptive Cards can create a simple, expressive means of describing card content.</span></span>

### <a name="when-in-doubt-keep-it-out"></a><span data-ttu-id="f9016-149">Im Zweifelsfall weglassen</span><span class="sxs-lookup"><span data-stu-id="f9016-149">When in doubt, keep it out</span></span>
<span data-ttu-id="f9016-150">Es ist einfacher, etwas nachträglich hinzuzufügen, als mit einem Fehler zu leben.</span><span class="sxs-lookup"><span data-stu-id="f9016-150">It is easier to add later then it is to live with a mistake.</span></span> <span data-ttu-id="f9016-151">Wenn wir darüber diskutiert haben, ob wir etwas hinzufügen sollten oder nicht, haben wir uns entschieden, es wegzulassen.  Es ist immer einfacher, eine Eigenschaft hinzuzufügen, als mit einer Vorgängerversion zu leben, von der wir uns wünschen, dass wir sie nicht unterstützen müssten.</span><span class="sxs-lookup"><span data-stu-id="f9016-151">If we found ourselves debating whether we should add something or not, we opted to leave it out.  It is always easier to add a property than to live with a legacy we wish we didn't have to support.</span></span>
