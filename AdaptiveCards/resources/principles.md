# <a name="motivations-and-principles"></a><span data-ttu-id="0dd32-102">Beweggründe und Prinzipien</span><span class="sxs-lookup"><span data-stu-id="0dd32-102">Motivations and Principles</span></span>

<span data-ttu-id="0dd32-103">Im Folgenden findest du die Beweggründe und Prinzipien, die die Entwicklung der adaptiven Karten prägen.</span><span class="sxs-lookup"><span data-stu-id="0dd32-103">Below are the motivations and principles govering the evolution of the Adaptive Card format.</span></span>

## <a name="motivations-behind-the-format"></a><span data-ttu-id="0dd32-104">Beweggründe für das Format</span><span class="sxs-lookup"><span data-stu-id="0dd32-104">Motivations behind the format</span></span>

<span data-ttu-id="0dd32-105">Anfang 2016 wurde verschiedenen Teams bei Microsoft (u. a. den Teams für Outlook, Windows und das Bot Framework) bewusst, dass sie alle etwas sehr Ähnliches („Karten“) wünschten und dass jedes Team unabhängig von den anderen seine eigene Lösung entwarf:</span><span class="sxs-lookup"><span data-stu-id="0dd32-105">In early 2016, several teams at Microsoft (including Outlook, Windows and the Bot Framework) came to the realization that they all wanted something extremely similar ("cards") and that each of them were designing their own solutions independently:</span></span>

- <span data-ttu-id="0dd32-106">Windows verfügte über ein eigenes Format mit Livekacheln und Benachrichtigungen.</span><span class="sxs-lookup"><span data-stu-id="0dd32-106">Windows had its own Live Tiles and Notifications format</span></span>
-  <span data-ttu-id="0dd32-107">Das Bot Framework verwendete eine Reihe vordefinierter Kartenvorlagen, aus denen Entwickler beim Senden von Botnachrichten auswählen konnten.</span><span class="sxs-lookup"><span data-stu-id="0dd32-107">The Bot framework was using a set of predefined card templates developers could choose from when sending Bot messages</span></span>
- <span data-ttu-id="0dd32-108">In Outlook kam ein eigenes MessageCard-Format für das Feature „Aktionen erfordernde Nachrichten“ zum Einsatz.</span><span class="sxs-lookup"><span data-stu-id="0dd32-108">Outlook was using its own MessageCard format for its Actionable Messages feature</span></span>

<span data-ttu-id="0dd32-109">Gleichzeitig wurde auch für Plattformen wie LINE, Facebook Messenger, Slack und viele weitere ein jeweils eigenes, proprietäres „Kartenformat“ definiert.</span><span class="sxs-lookup"><span data-stu-id="0dd32-109">At the same time, other platforms such as LINE, FaceBook Messenger, Slack and more were defining their own, proprietary "card" format.</span></span> <span data-ttu-id="0dd32-110">Also haben sich ein paar Microsoft-Mitarbeiter zusammengesetzt und damit begonnen, ein einzelnes, offenes Kartenformat sowie eine Reihe von SDKs zu definieren. Folgende Anforderungen sollten erfüllt werden:</span><span class="sxs-lookup"><span data-stu-id="0dd32-110">So a few Microsoft employees gathered up and started an effort to define a single, open card format and a set of SDKs that:</span></span>

- <span data-ttu-id="0dd32-111">Der Austausch von Karten zwischen Hosts sollte erleichtert werden.</span><span class="sxs-lookup"><span data-stu-id="0dd32-111">Would facilitate the interchange of cards between hosts</span></span>
- <span data-ttu-id="0dd32-112">Jeder Host sollte die Kontrolle über die Formatierung der Karten erhalten, um visuelle Konsistenz sicherzustellen.</span><span class="sxs-lookup"><span data-stu-id="0dd32-112">Would allow each host to keep control over the styling of cards to ensure visual consistency</span></span>
- <span data-ttu-id="0dd32-113">Mit einsatzbereiten SDKs sollte es Hostanwendungen erleichtert werden, Karten mit minimalem Aufwand anzuzeigen.</span><span class="sxs-lookup"><span data-stu-id="0dd32-113">Would make it easy for a host application to display cards with minimum effort via ready-to-use SDKs</span></span>
- <span data-ttu-id="0dd32-114">So sollte auch ein Mehrwert für Drittanbieter entstehen, damit das Format letztendlich branchenweit übernommen würde.</span><span class="sxs-lookup"><span data-stu-id="0dd32-114">And that would also provide value to third parties and eventually get adopted widely by the industry</span></span>

## <a name="principles-governing-adaptive-cards"></a><span data-ttu-id="0dd32-115">Die Prinzipien hinter adaptiven Karten</span><span class="sxs-lookup"><span data-stu-id="0dd32-115">Principles governing Adaptive Cards</span></span>

1.  <span data-ttu-id="0dd32-116">**Adaptive Karten sind ein _einfaches_ und _deklaratives_ Kartenformat.**</span><span class="sxs-lookup"><span data-stu-id="0dd32-116">**Adaptive Card is a _simple_ and _declarative_ card format**</span></span>

    1.  <span data-ttu-id="0dd32-117">Sie sind nicht als Ersatz oder Alternative für HTML oder XAML gedacht.</span><span class="sxs-lookup"><span data-stu-id="0dd32-117">It is not meant as an HTML or XAML replacement/alternative</span></span>
    2.  <span data-ttu-id="0dd32-118">Bei adaptiven Karten gibt es **kein CodeBehind**.</span><span class="sxs-lookup"><span data-stu-id="0dd32-118">There is **no "code behind"** with Adaptive Cards</span></span>
        1. <span data-ttu-id="0dd32-119">Kartenersteller können keinen benutzerdefinierten oder beliebigen Code in ihre Nutzlasten einbetten, daher muss ein Host für adaptive Karten niemals Drittanbietercode ausführen.</span><span class="sxs-lookup"><span data-stu-id="0dd32-119">Card authors cannot embed custom/arbitrary code with their payloads, and as a result an Adaptive Card host never needs to run third party code</span></span>
        2. <span data-ttu-id="0dd32-120">Dynamik und Interaktivität werden ausschließlich über deklaratives Markup erzielt.</span><span class="sxs-lookup"><span data-stu-id="0dd32-120">Dynamism and interactivity are achieved solely via declarative markup</span></span>
    3.  <span data-ttu-id="0dd32-121">So ist sichergestellt, dass der Aufwand für die Erstellung eines SDKs für adaptive Karten für eine neue Plattform in angemessenem Rahmen bleibt.</span><span class="sxs-lookup"><span data-stu-id="0dd32-121">This ensures that the effort necessary to build an Adaptive Card SDK for a new platform remains reasonable</span></span>

2.  <span data-ttu-id="0dd32-122">**Das Format der adaptiven Karten kann nicht die Verwendung einer bestimmten zugrunde liegenden Technologie vorschreiben.**</span><span class="sxs-lookup"><span data-stu-id="0dd32-122">**The Adaptive Card format can't impose the use of any particular underlying technology**</span></span>

    1.  <span data-ttu-id="0dd32-123">Das Format der adaptiven Karten basiert nicht auf JavaScript, C#, Python oder einer anderen Sprache.</span><span class="sxs-lookup"><span data-stu-id="0dd32-123">The Adaptive Card format does not rely on JavaScript, C#, Python or any other language</span></span>
    2.  <span data-ttu-id="0dd32-124">Es ist auch nicht von HTML, XAML oder einem anderen grafischen oder Benutzeroberflächenformat abhängig.</span><span class="sxs-lookup"><span data-stu-id="0dd32-124">Similarly it doesn't rely on HTML or XAML or any other graphic/UI framework</span></span>
    3.  <span data-ttu-id="0dd32-125">So können adaptive Karten nativ auf jeder Plattform gerendert, sofern ein Renderer vorhanden ist.</span><span class="sxs-lookup"><span data-stu-id="0dd32-125">This way, Adaptive Cards can be rendered natively on any platform for as long as a renderer exists</span></span>

3.  <span data-ttu-id="0dd32-126">**Das Format der adaptiven Karten ist eine _freigegebene Eigenschaft_.**</span><span class="sxs-lookup"><span data-stu-id="0dd32-126">**The Adaptive Card format is a _shared property_**</span></span>

    1.  <span data-ttu-id="0dd32-127">Das Format sowie die zugehörigen SDKs ist Open Source, und die Entwicklung erfolgt in der Community.</span><span class="sxs-lookup"><span data-stu-id="0dd32-127">Along with its SDKs, the format is to be open source and its evolution community driven</span></span>
    2.  <span data-ttu-id="0dd32-128">Daher ist das Format nicht im Besitz eines bestimmten Teams und wird von keinem Team gesteuert.</span><span class="sxs-lookup"><span data-stu-id="0dd32-128">The format is therefore not owned nor driven by any one team</span></span>

4.  <span data-ttu-id="0dd32-129">**Adaptive Karten sind nicht „nur zur Verwendung durch Microsoft“ konzipiert.**</span><span class="sxs-lookup"><span data-stu-id="0dd32-129">**The Adaptive Card format is not designed "just for Microsoft's use"**</span></span>

    1.  <span data-ttu-id="0dd32-130">Stattdessen erfüllt es die Anforderungen einer Vielzahl von Anwendungen und Anwendungsfällen.</span><span class="sxs-lookup"><span data-stu-id="0dd32-130">Instead it is designed to suit the needs of a wide variety of applications and use cases</span></span>

5.  <span data-ttu-id="0dd32-131">**Die _Adaptive Card Working Group_ steuert die Weiterentwicklung des Formats.**</span><span class="sxs-lookup"><span data-stu-id="0dd32-131">**The _Adaptive Card Working Group_ governs the evolution of the format**</span></span>

    1.  <span data-ttu-id="0dd32-132">Diese Arbeitsgruppe besteht aus einer Reihe von Microsoft-Mitarbeitern, die maßgeblich zum Erfolg des Formats beitragen.</span><span class="sxs-lookup"><span data-stu-id="0dd32-132">This working group is comprised of a set of Microsoft employees that are all deeply involved in the success of the format</span></span>
    2.  <span data-ttu-id="0dd32-133">Die Arbeitsgruppe trifft sich einmal in der Woche (zurzeit montags).     3.  <span data-ttu-id="0dd32-134">Die Arbeitsgruppe nutzt das Feedback aus der gesamten Community, einschließlich interner Microsoft-Teams, um zu entscheiden, wie sich das Format weiterentwickelt.</span><span class="sxs-lookup"><span data-stu-id="0dd32-134">The Working Group uses feedback from the community at large, including internal Microsoft teams, to decide how the format evolves</span></span>
        1. <span data-ttu-id="0dd32-135">Jeder kann sich durch Eröffnen von Issues in GitHub an der Arbeitsgruppe beteiligen (siehe unten).</span><span class="sxs-lookup"><span data-stu-id="0dd32-135">Anyone can participate in the working group through opening issues in GitHub (see below)</span></span>
        2. <span data-ttu-id="0dd32-136">Issues bzw. Featureanforderungen, die der tatsächlichen Nutzung des Formats entspringen (entweder als Host oder als Ersteller von Karten) haben den größten Einfluss auf die Zukunft des Formats.</span><span class="sxs-lookup"><span data-stu-id="0dd32-136">Issues/feature requests rooted in actual usage of the Adaptive Cards framework (either as a host or as a card author) have the most impact on the future of the format</span></span>
    4.  <span data-ttu-id="0dd32-137">Vorgeschlagene neue Features müssen folgende Anforderungen erfüllen, um von der Arbeitsgruppe berücksichtigt zu werden:</span><span class="sxs-lookup"><span data-stu-id="0dd32-137">To be approved by the Working Group, proposed new features:</span></span>
        1. <span data-ttu-id="0dd32-138">Sie müssen durch reale Anwendungsfälle gerechtfertigt sein.</span><span class="sxs-lookup"><span data-stu-id="0dd32-138">Must be justified by real life use cases</span></span>
        2. <span data-ttu-id="0dd32-139">Sie müssen eine funktionale Spezifikation aufweisen.</span><span class="sxs-lookup"><span data-stu-id="0dd32-139">Must have a functional specification</span></span>
    5.  <span data-ttu-id="0dd32-140">Genehmigte neue Features werden zum Backlog hinzugefügt und für vNext in Betracht gezogen.</span><span class="sxs-lookup"><span data-stu-id="0dd32-140">Approved new feature are added to the backlog and considered for vNext</span></span>
        1. <span data-ttu-id="0dd32-141">Neue Features werden anhand verschiedener Kriterien priorisiert. Hierzu gehören der Umfang der Szenarien, die durch dieses Feature ermöglicht werden, die Komplexität, die Verwaltungsfreundlichkeit und einige weitere.</span><span class="sxs-lookup"><span data-stu-id="0dd32-141">The criteria used to prioritize new features include the breadth of scenarios the feature enables, its complexity/maintainability and more</span></span>
        2. <span data-ttu-id="0dd32-142">Wenn du nicht sicher bist, lass es sein!</span><span class="sxs-lookup"><span data-stu-id="0dd32-142">When in doubt, keep it out!</span></span> <span data-ttu-id="0dd32-143">Es ist viel einfacher, ein gut konzipiertes Feature zu einem späteren Zeitpunkt einzuführen, als für immer mit einem Fehler zu leben.</span><span class="sxs-lookup"><span data-stu-id="0dd32-143">It's a lot easier to introduce a well designed feature later than to live with a mistake forever.</span></span>
    6.  <span data-ttu-id="0dd32-144">Alle neuen Features werden in allen unterstützten SDKs implementiert.</span><span class="sxs-lookup"><span data-stu-id="0dd32-144">All new features are implemented in all supported SDKs</span></span>
    7.  <span data-ttu-id="0dd32-145">Alle neuen Features werden dokumentiert und mit einer Testkarte verknüpft, die im Ordner mit Beispielen veröffentlicht wird.</span><span class="sxs-lookup"><span data-stu-id="0dd32-145">All new features are documented and associated with a test card published in the samples folder</span></span>
    8.  <span data-ttu-id="0dd32-146">Neue Versionen des Formats und der SDKs durchlaufen eine Betaphase.</span><span class="sxs-lookup"><span data-stu-id="0dd32-146">New versions of the format and SDKs go through a beta phase</span></span>
    9.  <span data-ttu-id="0dd32-147">Der Releasezeitplan für Schema- und SDK-Versionen einer adaptiven Karte basiert auf der Qualität, nicht auf einem bestimmten Datum.</span><span class="sxs-lookup"><span data-stu-id="0dd32-147">The release schedule for Adaptive Card schema and SDK versions is driven by quality, not date</span></span>

6.  <span data-ttu-id="0dd32-148">**Interoperabilität**</span><span class="sxs-lookup"><span data-stu-id="0dd32-148">**Interoperability**</span></span>
    1.  <span data-ttu-id="0dd32-149">Karten, die anhand eines dokumentierten Formats erstellt werden (also nicht mithilfe von hostspezifischen Erweiterungen), werden auf jedem Host ordnungsgemäß gerendert, der adaptive Karten unterstützt.</span><span class="sxs-lookup"><span data-stu-id="0dd32-149">Cards authored in accordance with the documented format (e.g. not using any host-specific extensions) will render properly in any Adaptive Card-capable host</span></span>
    2.  <span data-ttu-id="0dd32-150">Für diese Prinzipien gelten nur folgende Ausnahmen:</span><span class="sxs-lookup"><span data-stu-id="0dd32-150">The only exceptions to this principles are:</span></span>
        1.  <span data-ttu-id="0dd32-151">Einige Hosts erlauben möglicherweise keine Interaktivität und rendern daher weder Eingaben noch Aktionen.</span><span class="sxs-lookup"><span data-stu-id="0dd32-151">Some hosts might not allow interactivity, and will therefore not render inputs nor actions</span></span>
        2.  <span data-ttu-id="0dd32-152">Die Ausführung von Action.Submit-Aktionen erfolgt nur über den Host, und nicht alle Hosts verarbeiten notwendigerweise alle Action.Submit-Nutzlasten.</span><span class="sxs-lookup"><span data-stu-id="0dd32-152">Execution of Action.Submit actions is at the discretion of the host, and not all hosts will necessarily properly handle all Action.Submit payloads.</span></span> <span data-ttu-id="0dd32-153">Darüber hinaus unterstützen einige Hosts möglicherweise überhaupt keine Action.Submit-Aktionen.</span><span class="sxs-lookup"><span data-stu-id="0dd32-153">Furthermore, some hosts might not support Action.Submit at all</span></span>

7.  <span data-ttu-id="0dd32-154">**Das Format muss erweiterbar sein.**</span><span class="sxs-lookup"><span data-stu-id="0dd32-154">**The format needs to be extensible**</span></span>

    1.  <span data-ttu-id="0dd32-155">Hosts müssen die Möglichkeit haben, Unterstützung für benutzerdefinierte Elemente oder Aktionen hinzuzufügen, die die Funktionalität des Formats übersteigen.</span><span class="sxs-lookup"><span data-stu-id="0dd32-155">Hosts should have the freedom of adding support for custom elements or custom actions that go beyond what the format is capable of</span></span>
    2.  <span data-ttu-id="0dd32-156">Dies ist besonders wichtig bei Aktionen, da verschiedene Hosts nicht unbedingt den gleichen Satz Aktionen unterstützen.</span><span class="sxs-lookup"><span data-stu-id="0dd32-156">This is particularly important for actions, as various hosts don't necessarily support the same set of actions</span></span>
    3.  <span data-ttu-id="0dd32-157">Diese Hinzufügungen werden über den Host gesteuert.</span><span class="sxs-lookup"><span data-stu-id="0dd32-157">These additions are at the discretion of the host</span></span>
        1. <span data-ttu-id="0dd32-158">Sie sind keine *De-facto*-Hinzufügung zur Spezifikation von adaptiven Karten.</span><span class="sxs-lookup"><span data-stu-id="0dd32-158">They are not a *de facto* addition to the Adaptive Card specification</span></span>
        2. <span data-ttu-id="0dd32-159">Daher ist eine Nutzlast, die diese Karten verwendet, mit dem normalen Format der adaptiven Karten nicht kompatibel.</span><span class="sxs-lookup"><span data-stu-id="0dd32-159">As such, they make a payload that uses them incompatible with the mainstream Adaptive Card format</span></span>
        3. <span data-ttu-id="0dd32-160">Sie können jedoch der Arbeitsgruppe vorgestellt und als neue Features für eine zukünftige Version des Formats vorgeschlagen werden, wie in Punkt 5 beschrieben.</span><span class="sxs-lookup"><span data-stu-id="0dd32-160">They can however be presented to the Working Group and proposed as new features for a future version of the format, as described in point #5</span></span>

    1.  <span data-ttu-id="0dd32-162">Host-Apps erzwingen ihren Stil, daher sehen Karten so aus und verhalten sich so, als seien sie native Erweiterungen der App-Funktionalität.</span><span class="sxs-lookup"><span data-stu-id="0dd32-162">Host apps impose their styling so cards look and feel like they are native extensions of the app's experience</span></span>

9.  <span data-ttu-id="0dd32-164">**Für die beliebtesten Entwicklerplattformen werden SDKs bereitgestellt.**</span><span class="sxs-lookup"><span data-stu-id="0dd32-164">**SDKs will be provided for the most popular developer platforms**</span></span>

    1.  <span data-ttu-id="0dd32-165">SDKs vereinfachen das Rendern von Nutzlasten von adaptiven Karten auf jedem Host.</span><span class="sxs-lookup"><span data-stu-id="0dd32-165">SDKs make it easy to render Adaptive Card payloads in any host</span></span>
    2.  <span data-ttu-id="0dd32-166">So wird sowohl für Entwickler von Drittanbietern als auch für Microsoft-Teams ein möglichst einfacher Einstieg sichergestellt.</span><span class="sxs-lookup"><span data-stu-id="0dd32-166">This ensures the barrier to entry is as low as can be both for third party developers and Microsoft teams</span></span>
